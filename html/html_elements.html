HTML Елементи вебсторінки

HTML елемент зазвичай складається із тегу початкового і тегу кінцевого, а зміст вставляється поміж ними:

<tagname>Зміст розміщується тут...</tagname>

HTML елемент - це все від початкового тегу до кінцевого тегу:

<p>Мій перший параграф.</p>
Початковий тег Зміст елементу Кінцевий тег
<h1> Мій перший заголовок </h1>
<p> Мій перший параграф. </p>
<br>    

HTML елементи без змісту називаються порожніми елементами. Порожні елементи не мають кінцевого тегу, наприклад, як елемент <br> (який вказує на розрив рядка).

Вкладені HTML елементи

HTML елементи можуть бути вкладеними (елементи можуть містити елементи).

Всі документи HTML складаються з вкладених елементів HTML.

Цей приклад містить чотири елементи HTML:

Приклад

<!DOCTYPE html>
<html>
<body>

<h1>Мій перший заголовок</h1>
<p>Мій перший параграф.</p>

</body>
</html>

Пояснення прикладу

Елемент <html> визначає весь документ.

У нього є початковий тег <html> і кінцевий тег </html>.

Всередині <html> елементу знаходиться елемент <body>.

<html>
<body>

<h1>Мій перший заголовок</h1>
<p>Мій перший параграф.</p>

</body>
</html>

Елемент <body> визначає тіло документа.

У нього є початковий тег <body> і кінцевий тег </body>.

Всередині елемента <body> є два інших HTML елемента: <h1> та <p>.

<body>

<h1>Мій перший заголовок</h1>
<p>Мій перший параграф.</p>

</body>

Елемент <h1> визначає заголовок.

У нього є початковий тег <h1> і кінцевий тег </h1>.

Елемент зміст це: Мій перший заголовок.

<h1>Мій перший заголовок</h1>

Елемент <p> визначає параграф.

У нього є початковий тег <p> і кінцевий тег </p>.

Елемент зміст це: Мій перший параграф.

<p>Мій перший параграф.</p>

Не забувайте про кінцевий тег

Деякі елементы HTML будуть відображатись правильно, навіть якщо ви забудете кінцевий тег:

Приклад

<html>
<body>

<p>Це параграф
<p>Це параграф

</body>
</html>
Спробуйте самі »

Наведений вище приклад працює в усіх браузерах, оскільки закриваючий тег вважається не обов’язковим.

Ніколи не покладайтеся на це. Це може призвести до несподіваних результатів і / або помилок, якщо ви забудете кінцевий тег.

Порожні HTML елементи

HTML елементи без вмісту називаються порожніми елементами.

Елемент <br> - це порожній елемент без закриваючого тегу (тег <br> визначає обрив рядка):

Приклад

<p>Це параграф<br> з обривом рядка.</p>
Спробуйте самі »

Порожні елементи можуть бути "закриті" у відкриваючому тезі наступним чином: <br />.

HTML5 не вимагає закриття порожніх елементів. Але якщо вам потрібна більш сувора перевірка або якщо ви хочете, щоб ваш документ читався XML-аналізаторами, ви повинні правильно закрити всі HTML елементи.

HTML не чутливий до регістру

HTML-теги не чутливі до регістру: <P> означає те ж саме, що і <p>.

Стандарт HTML5 не вимагає написання тегів в нижньому регістрі, але W3C рекомендує використовувати літери в нижньому регістрі при написанні HTML тегів, і вимагає літери в нижньому регістрі для більш суворих типів документів, таких як XHTML.

В W3Schools завжди використовують написання тегів в нижньому регістрі.
