Робота з файлами

«Файл» – це основна одиниця того, що редагується в Notepad++, але цей термін насправді охоплює кілька пов’язаних понять. По-перше, «файл» – це послідовність байтів, що зберігаються на диску або іншому носії інформації та доступ до яких здійснюється через файлову систему вашого комп’ютера; і, як на зло, якщо документ, який ви редагуєте, ніколи не зберігався у файловій системі, технічно він не є файлом, хоча зазвичай цей термін застосовується і до незбережених документів. «Документ» стосується тексту, що редагується, незалежно від того, чи це новий, незбережений документ, чи це документ, який раніше був збережений як файл у файловій системі. І, нарешті, у Notepad++ кожен документ представлений у вкладці в одному з двох Views інтерфейсу користувача Notepad++, які є графічними контейнерами, що Notepad++ використовує для маніпулювання файлами та іншими документами, хоча багато користувачів також вважають елемент інтерфейсу користувача «файлом».

Меню File / Файл

Меню File / Файл містить багато звичайних дій з файлами, які можна знайти в більшості програм. Notepad++ також містить інші користувацькі дії, які корисно розуміти.

  • New / Новий: Створює новий документ і відображає цей порожній документ у вкладці активного перегляду.
  • Open / Відкрити: Відкриває існуючий файл із файлової системи та відображає цей документ у вкладці активного перегляду.
  • Відкрити папку-контейнер >:
    • Explorer / Провідник: Відкриває екземпляр Провідника Windows, починаючи з папки (також відомої як «каталог»), яка містить активний файл.
    • cmd: Відкриває екземпляр cmd.exe системи Windows, починаючи з папки, яка містить активний файл.
    • Folder as Workspace / Папка як робоча область: Відкриває папку активного файлу на панелі «Папка як робоча область» програми Notepad++.
  • Open in Default Viewer / Відкрити у програмі перегляду за замовчуванням: Це використовуватиме «асоціацію типу файлу» Windows для активного файлу (на основі розширення файлу).
    • Це еквівалентно подвійному клацанню на файлі в Провіднику Windows.
    • Якщо документ ніколи не зберігався як файл, ця дія буде неактивною, і пункт меню не можна буде використовувати.
    • Якщо тип файлу не має пов’язаної дії за замовчуванням у Windows, ця дія буде неактивною, і пункт меню не можна буде використовувати.
    • Якщо тип файлу пов’язаний з Notepad++ і, таким чином, просто повторно відкриває файл у Notepad++, ця дія буде неактивною, і пункт меню не можна буде використовувати.
  • Open Folder as Workspace / Відкрити папку як робочу область: Відкриває папку активного файлу на панелі «Папка як робочий простір» у Notepad++.
  • Reload from Disk / Перезавантажити з диска: Це повторно зчитає байти файлу з файлової системи, замінивши поточний вміст документа, що редагується, останньою збереженою версією (і таким чином втратить будь-які зміни, які могли бути внесені в Notepad++).
    • Якщо у вас увімкнено параметр Settings > Preferences > MISC > File Status Auto-detection turned on, Notepad++ може автоматично виконати еквівалент цієї команди, без необхідності використовувати цей пункт меню.
    • Будь-які закладки або стилі токенів, які ви встановили раніше, будуть очищені.
    • Будь-яку історію змін, записану для файлу, буде скинуто, і файл відображатиметься без кольорів полів історії змін.
  • Save / Зберегти: Зберігає поточний документ у файловій системі, використовуючи те саме ім'я файлу.
    • Якщо з документом наразі не пов’язано жодного файлу, ця дія буде неактивною, і пункт меню не можна буде використовувати. Натомість скористайтеся командою File > Save As (Файл > Зберегти як).
  • Save As / Зберегти як: Зберігає поточний документ у файловій системі, запитуючи нове ім'я файлу.
    • Якщо документ раніше було збережено у файлі, це створить нову копію файлу, і Notepad++ продовжить роботу з документом, пов'язаним з новим файлом, залишивши старий файл із вмістом, який було збережено востаннє.
  • Save a Copy As / Зберегти копію як: Зберігає поточний документ у файловій системі, запитуючи нове ім'я файлу для використання..
    • Новий файл буде збережено, але Notepad++ продовжить працювати з документом, пов’язаним з оригінальним ім’ям файлу. Таким чином, ця дія фактично створює резервну копію файлу, але дозволяє продовжувати роботу з попереднім файлом, тоді як функція «Зберегти як» використовує оригінал як резервну копію та дозволяє продовжувати роботу з новим файлом.
    • Зазвичай під час виконання цієї команди новостворений файл не відкривається на вкладці редагування Notepad++. Щоб відкрити новий файл як частину процесу, утримуйте клавішу Shift, натискаючи кнопку Save / Зберегти в діалоговому вікні Save As / Зберегти як.
  • Save All / Зберегти все: Зберігає всі документи, які наразі відкриті в одному з двох подань Notepad++.
  • Rename / Перейменувати: Запитує нову назву для поточного документа.
    • Якщо документ пов’язаний із файлом, Notepad++ відкриє стандартне діалогове вікно Windows для збереження файлу під новим ім’ям у файловій системі, і старе ім’я файлу більше не існуватиме.
    • Якщо документ є новим, незбереженим документом, ім’я, що відображається на вкладці для поточного документа, зміниться. Якщо пізніше ви збережете як, воно використовуватиметься як запропоноване ім’я за замовчуванням у діалоговому вікні «Зберегти як».
  • Close / Закрити: Закриває активний документ, запитуючи на збереження, якщо він був змінений з моменту відкриття або створення.
  • Close All / Закрити все: Закриває всі документи, відкриті в поточному екземплярі Notepad++, і пропонує зберегти зміни, якщо один або декілька файлів було змінено з моменту їх відкриття або створення.
    • У діалоговому вікні запиту Yes / Так збереже файл, No / Ні закриє його без збереження (ви втратите всі зміни), Cancel / Скасувати зупинить закриття всіх файлів у Notepad++.
    • Якщо запитується кілька файлів, ви також отримаєте Yes to All / Так для всіх, щоб зберегти всі змінені файли, і No to All / Ні для всіх, щоб втратити зміни з усіх змінених файлів.
  • Закрити кілька документів >:
    • Закрити всі, крім активного документа: Закриває всі відкриті документи, крім документа, який активно редагується.
    • Закрити всі, крім закріплених документів: Закриває всі відкриті документи, якщо їхня вкладка не закріплена.
    • Закрити всі ліворуч: Закриває всі відкриті документи, що знаходяться ліворуч від активного документа на панелі вкладок поточного перегляду.
    • Закрити всі праворуч: Закриває всі відкриті документи, що знаходяться праворуч від активного документа на панелі вкладок поточного перегляду.
    • Закрити всі без змін: Закриває всі відкриті документи, які не редагувалися з моменту їх останнього відкриття або створення.
  • Move to Recycle Bin / Перемістити до кошика: Закриває активний документ і переміщує базовий файл до кошика Windows. Якщо документ не має базового файлу, ця дія буде неактивною, і пункт меню не можна буде використовувати.
  • Load Session… / Завантажити сеанс…: Завантажує всі файли зі збереженого файлу сеансу.
  • Save Session… / Зберегти сеанс…: Зберігає поточний сеанс (список відкритих файлів) до файлу сеансу.
  • Print / Друк: Відкриває діалогове вікно друку для друку файлу, як описано в розділі «Друк».
  • Print Now / Друкувати зараз: Друкує активний файл без відображення діалогового вікна друку, використовуючи принтер за замовчуванням та інші налаштування діалогового вікна друку за замовчуванням.
  • Recent Files / Недавні файли >: Список нещодавно закритих файлів, які редагував Notepad++.
    • Якщо для параметра [Settings > Preferences > Recent Files History] налаштовано використання підменю, останні файли будуть у цьому підменю, інакше вони будуть у головному меню «Файл».
    • Клацання на записі файлу відкриє цей файл.
    • Restore Recent Closed File / Відновити нещодавно закритий файл: Відкриває файл, який було закрито останнім і знаходиться у верхній частині списку нещодавно відкритих файлів.
    • Open All Recent Files / Відкрити всі нещодавні файли: Відкриває всі файли зі списку.
    • Empty Recent Files List / Очистити список останніх файлів: Видаляє всі файли зі списку останніх файлів.
  • Exit / Вихід: Закриває програму Notepad++.
    • Якщо у вас встановлено прапорець Settings > Preferences > Backup ☑ Remember current session for next launch, Notepad++ збереже поточний список файлів у сеансі. Єдині файли, які будуть запам’ятані для наступного запуску Notepad++, будуть ті, що наразі відкриті в Notepad++.
    • Якщо у вас не встановлено прапорець Settings > Preferences > Backup ☐ Enable session snapshot and periodic backup, Notepad++ запропонує вам, чи зберігати будь-які змінені файли, дотримуючись тих самих правил, що й для параметра «Закрити все», описаного вище. Якщо у вас встановлено прапорець для цього параметра, незбережені документи також будуть включені до файлу сеансу, навіть якщо вони не мають базового файлу.

Фільтри типів файлів

Для діалогового вікна Зберегти як та подібних є випадаюче поле Зберегти як тип. Для діалогового вікна Відкрити (та подібних) є випадаюче поле типу файлу праворуч від запису Ім'я файлу. Ці випадаючі списки служать для фільтрації списку файлів у діалоговому вікні на основі типу файлу. Список типів файлів фільтра включатиме всі мови в меню Мова та розширення для цих типів, як визначено в Конфігураторі стилів). Починаючи з версії 8.7, фільтр також включатиме різні типи файлів та розширення Визначена користувачем мова.

Для діалогового вікна «Зберегти як» прапорець ☐ Append extension «Додати розширення», описаний у розділі «Перемикачі налаштувань в інших діалогових вікнах», дозволить діалоговому вікну автоматично додавати перше розширення у списку для активного фільтра. Якщо вам потрібне інше розширення для цього типу або якщо цей параметр не позначено, ви завжди можете ввести назву файлу з розширенням, щоб явно визначити потрібне розширення для файлу, який ви наразі зберігаєте. (Тому зазвичай виконання фільтра Зберегти як для C++ автоматично додає розширення .cpp; але якщо ви хочете, щоб це було .cxx або .h, ви можете просто ввести це як частину запису Ім'я файлу, а не дозволяти Notepad++ автоматично додавати розширення.)

Альтернативи меню «Файл»

Багато дій з меню «Файл» мають кілька способів доступу, як описано в Інтерфейсі користувача та в інших відповідних розділах Посібника користувача.

Наприклад,

  • File > Open має багато способів досягти одного й того ж:
    • Сполучення клавіш за замовчуванням Ctrl+N.
    • На панелі інструментів є значок «Створити».
    • У рядку меню може бути значок + для створення нового файлу.
    • Ви можете двічі клацнути вільне місце на панелі вкладок.
    • Ви можете скористатися контекстним меню значка системного трея, яке викликається клацанням правою кнопкою миші.
  • File > Close також має багато способів досягти того ж самого:
    • Сполучення клавіш за замовчуванням Ctrl+W.
    • На панелі інструментів є значок «Закрити».
    • У рядку меню може бути значок X для закриття активного файлу.
    • У розділі User Interface > Tabs пояснюються ще три пункти:
      • Ви можете натиснути X поруч із назвою вкладки, щоб закрити її, якщо встановлено параметр Settings > Preferences > General > Tab Bar > Show close button.
      • Ви можете двічі клацнути на заголовку вкладки, щоб закрити її, якщо встановлено параметр Settings > Preferences > General > Tab Bar > ☐ Double click to close document.
      • Ви можете клацнути середньою кнопкою миші на заголовку вкладки, щоб закрити її.
    • Ви можете скористатися контекстним меню назви вкладки, що випадає після клацання правою кнопкою миші.

Загалом, зайвим є пояснювати всі альтернативні методи в кількох місцях Посібника, тому, якщо ви шукаєте спосіб виконати дію з меню «Файл» в якомусь іншому місці, ви можете скористатися функцією Find in Page вашого браузера (часто Ctrl+F або подібною), щоб знайти потрібну вам дію на сторінці Інтерфейс користувача. А вивчення цієї сторінки дозволить вам ознайомитися з різними місцями, де можна знайти такі об'єкти.

Друк

Дія File > Print відкриє стандартне діалогове вікно друку Windows, в якому ви можете вибрати принтер і надіслати текст на вибраний принтер. Зазвичай друкується весь документ, але ви можете використовувати діалогове вікно друку, щоб вибрати лише певні сторінки; якщо у вас є активний вибір у редакторі, буде надруковано лише вибраний текст.

На друк тексту впливають налаштування Settings > Preferences > Print, де ви можете вирішити, чи включати номери рядків у роздруківку, ширину полів, додатковий текст для друку у верхньому та/або нижньому колонтитулах кожної сторінки, а параметри кольору визначатимуть, як кольори вашої теми конфігуратора стилів будуть поширюватися на друкований вивід. Ви як користувач повинні вирішити, які налаштування друку найкраще відповідають вашим потребам і найкраще виглядатимуть на вибраному вами друкованому носії. Зверніть увагу, що з будь-якими темами з кольоровим фоном деякі з цих кольорів будуть надруковані на фоні друкованої сторінки, якщо ви не оберете один із параметрів кольору, який не друкує колір фону; але з темнішими темами, які використовують світлий текст на темному фоні, якщо ви не налаштуєте друк кольору фону, то на білому папері буде дуже світлий текст. Якщо у вас є принтер, який дозволяє друкувати у форматі PDF, це гарний спосіб побачити, як насправді виглядатиме роздруківка без використання паперу та чорнил (а якщо у вас немає PDF-принтера, ваша улюблена пошукова система покаже вам безліч комерційних та безкоштовних PDF-принтерів, якщо ви пошукаєте щось на кшталт «PDF-принтер Windows» або «друк у PDF»).
