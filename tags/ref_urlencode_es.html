URL (Uniform Resource Locator): localizador uniforme de recursos

Los navegadores web solicitan páginas de los servidores web mediante una URL.

La URL es la dirección de una página web, como: https://www.w3schools.com/.

Codificación de URL (codificación porcentual)

La codificación de URL convierte caracteres a un formato que se puede transmitir a través de Internet.

Las URL sólo se pueden enviar a través de Internet utilizando el juego de caracteres ASCII.

Dado que las URL a menudo contienen caracteres fuera del conjunto ASCII, la URL debe convertirse a un formato ASCII válido.

La codificación de URL reemplaza los caracteres ASCII no seguros con un "%" seguido de dos dígitos hexadecimales.

Las URL no pueden contener espacios. La codificación de URL normalmente reemplaza un espacio con un signo más (+) o con %20.

Pruébelo usted mismo

Si hace clic en el botón "Enviar" a continuación, el navegador codificará la URL de la entrada antes de enviarla al servidor. Una página en el servidor mostrará la entrada recibida.

Pruebe con otra entrada y haga clic en Enviar nuevamente.

Funciones de codificación de URL

En JavaScript, PHP y ASP, existen funciones que se pueden utilizar para codificar una cadena mediante URL.

PHP tiene la función rawurlencode() y ASP tiene la función Server.URLEncode().

En JavaScript, puedes utilizar la función encodeURIComponent().

Haga clic en el botón "Codificar URL" para ver cómo la función JavaScript codifica el texto.

Nota: La función JavaScript codifica el espacio como %20.

Referencia de codificación ASCII

Su navegador codificará la entrada, de acuerdo con el conjunto de caracteres utilizado en su página.

El juego de caracteres predeterminado en HTML5 es UTF-8.

Los caracteres de control no tienen nada que hacer dentro de una URL.
