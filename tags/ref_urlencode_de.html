URL (Uniform Resource Locator) - Einheitlicher Ressourcenfinder

Webbrowser fordern Seiten von Webservern mithilfe einer URL an.

Die URL ist die Adresse einer Webseite, etwa: https://www.w3schools.com/.

URL-Kodierung (prozentuale Kodierung)

URL-Kodierung wandelt Zeichen in ein Format um, das über das Internet übertragen werden kann.

URLs können nur mit dem ASCII-Zeichensatz über das Internet gesendet werden.

Da URLs oft Zeichen außerhalb des ASCII-Satzes enthalten, muss die URL in ein gültiges ASCII-Format konvertiert werden.

URL-Kodierung ersetzt unsichere ASCII-Zeichen durch ein „%“, gefolgt von zwei hexadezimalen Ziffern.

URLs dürfen keine Leerzeichen enthalten. Bei der URL-Codierung wird ein Leerzeichen normalerweise durch ein Pluszeichen (+) oder durch %20 ersetzt.

Probieren Sie es selbst aus

Wenn Sie unten auf die Schaltfläche „Senden“ klicken, verschlüsselt der Browser die Eingabe per URL, bevor sie an den Server gesendet wird. Eine Seite auf dem Server zeigt die empfangene Eingabe an.

Versuchen Sie es mit einer anderen Eingabe und klicken Sie erneut auf „Senden“.

URL-Kodierungsfunktionen

In JavaScript, PHP und ASP gibt es Funktionen, die zum URL-Codieren einer Zeichenfolge verwendet werden können.

PHP verfügt über die Funktion rawurlencode() und ASP über die Funktion Server.URLEncode().

In JavaScript können Sie die Funktion encodeURIComponent() verwenden.

Klicken Sie auf die Schaltfläche „URL-Kodierung“, um zu sehen, wie die JavaScript-Funktion den Text kodiert.

Hinweis: Die JavaScript-Funktion kodiert Speicherplatz als %20.
